Ballast
‚Ä¢ Pages that don't want to be found for anything (i.e. all pages except

Targeted Pages and the Startpage) are ballast from SEO point of view.

‚Ä¢ Ballast should either not exist.
‚Ä¢ "HTTP 404 File not Found" for non-existing pages
‚Ä¢ "HTTP 410 Gone" for non-existent pages
‚Ä¢ Or not be indexed.
‚Ä¢ <meta name="robots" content="noindex">

Typical Ballast
‚Ä¢ Error Pages -> HTTP 404
‚Ä¢ Non-existent pages -> HTTP 404
‚Ä¢ Search Pages -> noindex
‚Ä¢ Pagination Pages -> noindex
‚Ä¢ Navigation Pages -> noindex
‚Ä¢ SEO Interlinking Pages -> noindex

‚Ä¢ If only a small amount of ballast pages exist on the website, simply ignore them. The problem starts when the main
part of the pages on large websites are ballast pages and hinders the pages that you care about to get crawled and
indexed.

‚Ä¢ But: Non-existing pages should always return HTTP 404 File not Found, otherwise you will spam yourself very
fast.

Edge Cases but not Ballast
‚Ä¢ Imprint, Terms and Conditions, Contact are not Ballast Pages
‚Ä¢ But "Branded Targeted Pages‚Äù
